# RegioBuild

[English version](README_EN.md)

**Проект для сравнения региональных строительных нормативов (РНГП/ТСН)
субъектов Российской Федерации** — с опорой на федеральный фон
(СП 42.13330.2016 и точечные выдержки 123-ФЗ / СанПиН).

Telegram здесь — демонстрационный клиент. Ядро — **RAG + LLM-агент** с
обязательной привязкой существенных тезисов к пункту нормативного акта.
API можно развернуть отдельно и вызвать из другого приложения.

## Статус и контекст

Сразу по существу: **текущая версия не претендует на законченный
коммерческий продукт.** Работают Telegram-бот и API, программная сверка
цитат с retrieved-фрагментами, индекс по пяти субъектам — но до устойчивого
сервиса ещё путь: расширение корпуса, муниципальные ПЗЗ, редкие формулировки
запросов, сопровождение данных. Всё собрано **в одиночку**.

Я не «наткнулся на проблему рынка и недоделал стартап». У меня юридическое
образование — бакалавриат и магистратура по юриспруденции (УрГЮУ), за плечами
практика в правовой сфере и опыт в продажах; разрозненность регионального
регулирования для меня предмет знакомый. Когда занялся ML, захотелось
проверить, можно ли собрать по этой задаче рабочий retrieval-контур с
проверяемыми ссылками на НПА. Цель изначально была исследовательско-инженерная,
а не вывод продукта в продажу; для серьёзной доработки нужна команда.

**О истории коммитов.** Долгое время пайплайн, векторный индекс и агент
развивались и тестировались **локально** (API, pytest, eval — без Telegram).
Бота в BotFather зарегистрировал недавно (дата создания там видна), поэтому
публичная история git в основном отражает выкладку и клиентскую обвязку.
Это не разработка «за три дня»: ядро и корпус НПА собирались раньше, просто
без удалённого репозитория и без мессенджера.

## О задаче

У каждого субъекта РФ свой акт о нормативах градостроительного проектирования.
Плотность регулирования неоднородна: в одном регионе по автомойке может быть
жёсткая норма машино-мест, в другом — пробел и опора на федеральный каркас.
Юристу и проектировщику это знакомо: часы ручной сверки постановлений и
приказов, риск пропустить существенное условие размещения.

RegioBuild — не диалог «поболтать про стройку», а попытка дать
**проверяемую справку**: номер пункта и уровень регулирования
(региональный / федеральный).

### Предметный скоуп (волна 1)

Регионы в индексе: Московская область, Краснодарский край, Свердловская и
Новосибирская области, Республика Татарстан. Федеральный слой: СП 42 и
кураторские выдержки 123-ФЗ / СанПиН.

Типы объектов: автомойка, автосервис, АЗС, склад, ТЦ/магазин, офис,
общепит, гостиница, медцентр, производство.

Это не весь классификатор Минстроя и не полный свод СП РФ — осознанно узкий
срез под типовые вопросы коммерческого размещения.

## Функциональность

1. **Требования в одном регионе** — обзор по типу объекта капитального
   строительства с разделением регионального и федерального уровней.
2. **Сравнение двух регионов** — различия и совпадения с указанием пунктов.

Номер пункта из ответа модели сверяется с найденными фрагментами индекса;
при отсутствии опоры предпочтителен отказ, а не выдуманная норма.

## Архитектура и стек

| Слой | |
|------|--|
| Язык | Python 3.11 |
| Backend | FastAPI (`/info`, `/compare`, `/health`, `/metrics`) |
| Клиент | aiogram 3 |
| Оркестрация | LangGraph |
| Retrieval | sentence-transformers (multilingual MiniLM), Chroma |
| Классификация категорий | scikit-learn (TF-IDF + LogisticRegression) |
| LLM | GigaChat в проде; в коде также YandexGPT |
| Данные | SQLAlchemy + Alembic, disk LLM-cache |
| Качество | pytest, Recall@k / MRR |
| Infra | Docker, GitHub Actions, Prometheus, Sentry |

Один Docker-образ, роль через `SERVICE_ROLE=api|bot`.

Схема пайплайна: [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## Инженерные узкие места

- **Ingestion НПА** — разнородные форматы, табличный мусор, ложная нумерация
  пунктов, неравномерная плотность корпусов по субъектам.
- **Retrieval и grounding** — язык пользователя ≠ канцелярит нормы; без
  фильтра usable-чанков и сверки citation модель склонна галлюцинировать
  номера статей.
- **Эксплуатация на ограниченной RAM** — cold start embedder, раздельные
  роли api/bot. Пилот: Bothost + GigaChat.
  [`docs/BOTHOST_CHECKLIST.md`](docs/BOTHOST_CHECKLIST.md).

## Локальный запуск

```bash
python -m venv venv
venv\Scripts\activate          # Linux/Mac: source venv/bin/activate
pip install -r requirements.txt
copy .env.example .env         # Linux/Mac: cp .env.example .env
```

Ключи LLM и токен бота — в `.env`:

```bash
alembic upgrade head
python -m app.ingestion.pipeline        # при необходимости пересобрать корпус
python -m app.embeddings.build_index
python -m app.classifier.train
python -m scripts.ingest_curated

uvicorn app.api.main:app --reload
python -m app.bot.main                  # второй терминал
```

Или: `docker compose up --build`. Postgres: `docker-compose.postgres.yml`.
