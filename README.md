# RegioBuild

[English version](README_EN.md)

**Проект для сопоставления региональных нормативов градостроительного
проектирования (РНГП/ТСН) субъектов Российской Федерации** с учётом
федерального регулирования (СП 42.13330.2016, выдержки из Федерального закона
от 22.07.2008 № 123-ФЗ и санитарно-эпидемиологических правил и нормативов).

Telegram-клиент служит демонстрационным интерфейсом. Архитектурное ядро —
сервис FastAPI и RAG-агент (LangGraph): существенные утверждения в ответе
подлежат привязке к конкретному пункту нормативного акта. Программный
интерфейс может быть развёрнут независимо от мессенджера.

## Назначение

Правовое регулирование размещения объектов капитального строительства в
субъектах Российской Федерации характеризуется значительной фрагментацией:
содержание и плотность региональных нормативов градостроительного
проектирования существенно различаются. Сопоставление актов субъектов и
федеральных норм вручную трудоёмко и сопряжено с риском пропуска
существенных требований.

RegioBuild формирует **справочный обзор** с указанием номера пункта и уровня
регулирования (региональный / федеральный). Результат не подменяет
юридическую консультацию, проектную документацию и профессиональную оценку
допустимости размещения.

## Текущее состояние

Реализованы демонстрационный Telegram-клиент, HTTP API, векторный индекс по
пяти субъектам Российской Федерации, программная верификация цитат относительно
извлечённых фрагментов корпуса. Версия носит характер **рабочего прототипа**:
расширение предметного охвата, включение муниципальных ПЗЗ, устойчивость на
нетипичных формулировках запросов и сопровождение корпуса требуют дальнейшей
разработки — в том числе силами команды, а не одного автора.

Разработка и тестирование ядра (ingestion, индекс, агент, API, оценочные
прогоны) длительное время велись локально. Публикация репозитория и подключение
Telegram-клиента относятся к более позднему этапу; краткая публичная история
git отражает прежде всего выкладку и клиентский слой, а не полный цикл
создания решения.

## Возможности

1. Обзор требований для объекта капитального строительства в одном субъекте
   Российской Федерации — с разделением регионального и федерального уровней.
2. Сопоставление требований двух субъектов — различия и совпадения с указанием
   пунктов актов.

Номер пункта, предложенный языковой моделью, сопоставляется с фрагментами,
полученными из индекса; при отсутствии проверяемой опоры система
воздерживается от генерации нормы.

## Охват индекса

Субъекты Российской Федерации, представленные в текущем индексе: Московская
область, Краснодарский край, Свердловская область, Новосибирская область,
Республика Татарстан.

Федеральный слой: СП 42.13330.2016; кураторские фрагменты Федерального закона
№ 123-ФЗ и СанПиН в части, необходимой для фонового сопоставления.

Индекс не претендует на исчерпывающий охват классификатора объектов и полного
свода сводов правил Российской Федерации.

## Стек

| Слой | Состав |
|------|--------|
| Язык | Python 3.11 |
| Backend | FastAPI (`/info`, `/compare`, `/health`, `/metrics`) |
| Клиент | aiogram 3 |
| Оркестрация | LangGraph |
| Retrieval | sentence-transformers (multilingual MiniLM), Chroma |
| Классификация | scikit-learn (TF-IDF + LogisticRegression) |
| LLM | GigaChat (прод); в коде также YandexGPT |
| Данные | SQLAlchemy, Alembic, дисковый кэш ответов LLM |
| Качество | pytest; Recall@k, MRR |
| Инфраструктура | Docker, GitHub Actions, Prometheus, Sentry |

Единый Docker-образ; роль процесса задаётся переменной `SERVICE_ROLE=api|bot`.

Описание пайплайна: [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## Локальный запуск

```bash
python -m venv venv
venv\Scripts\activate          # Linux/Mac: source venv/bin/activate
pip install -r requirements.txt
copy .env.example .env         # Linux/Mac: cp .env.example .env
```

Учётные данные LLM и токен бота указываются в `.env`:

```bash
alembic upgrade head
python -m app.ingestion.pipeline
python -m app.embeddings.build_index
python -m app.classifier.train
python -m scripts.ingest_curated

uvicorn app.api.main:app --reload
python -m app.bot.main
```

Альтернатива: `docker compose up --build`. Вариант с Postgres —
`docker-compose.postgres.yml`. Операционные замечания по выкладке:
[`docs/BOTHOST_CHECKLIST.md`](docs/BOTHOST_CHECKLIST.md).

## Тестирование

```bash
pytest
python -m app.eval.retrieval_eval
python -m app.eval.answer_eval
python -m scripts.audit_corpus
```

Непрерывная интеграция (GitHub Actions) выполняет облегчённый набор проверок
без torch/chroma. Smoke после развёртывания: `scripts/smoke_wave1_prod.py`.
